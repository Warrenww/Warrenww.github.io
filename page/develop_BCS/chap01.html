前言

工欲善其事，必先利其器，第一章我們輕鬆一點，先把基礎建設做好，架出一個node js的server，還有基本的首頁， 並確立兩者間可以用socket io這個模組來溝通。如果你完全沒有聽過node、npm、react甚至連網頁都沒寫過也沒關係， 我會盡量把我當年(?)的學習歷程與過程中可能會有的問題寫下來。

本章目標

  • 安裝node js

  • 架設node js server可以存取網頁

安裝node js

前往node js的官網即可下載node js，按照指示安裝完成之後，就可以開始使用node js了。 首先打開終端機輸入: 快速打開終端機的方法
按下windows+R開啟執行視窗，輸入cmd，即可叫出終端機(此時路徑在 C:/Users/{username}之下 ) 

        
          node -v

指令來檢查node js是否有安裝好，如果順利安裝好的話，就會顯示目前安裝的node版本，筆者目前用的是v10.14.2版，node js在本專案中主要擔任後端代理資料庫的腳色， 之後我們會連接firebase作為資料庫，但因為免費版本有存取次數的限制，所以必須先把資料cache到後端主機的記憶體中，才可以無限制地讓使用者存取資料。 除了快取的功能外，後端也作為一些資料的預處理節點，將結果先行簡化，以減少前端的運算負擔與流量負擔。

安裝node模組

node js因為使用率非常的高，所以生態的發展也相對的成熟，套件模組的種類數量都非常多樣，而其模組管理系統，node package manager(NPM)， 會隨者node js一併安裝，同上一段的方法，在終端機輸入已下指令即可檢查npm是否有正確的安裝，並查看版本號: 

        
          npm -v

筆者目前使用的版本是6.4.1版。確認好npm有正確安裝後，就可以用npm來下載安裝模組了。 首先要安裝的是最重要的React。React是由fb開發的開源javascript框架，有別於傳統的Html + Css + JS的網頁結構， React主打完全使用javascript來渲染網頁，並採取重複利用component的思維，來減少重複造輪，另外也用了OOP (Object-oriented programming)的觀念加強維護性。
